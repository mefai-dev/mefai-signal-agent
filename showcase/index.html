Autonomous trading signals, settled on BNB Chain.

02 · The Protocol✕
Funded first. Then delivered. Always verifiable.

Every job runs the same ERC-8183 lifecycle. The agent never prices with the LLM and never signs from a tool. Pricing is fixed, signing is deterministic code.

05 · BNB Chain Rails✕
The stack

◆ BNB Smart Chain

Identity, jobs, escrow and settlement all live on BSC. The agent's wallet is the single on-chain signer.

◈ ERC-8004 Identity

The agent registers a discoverable, verifiable on-chain identity. Buyers resolve it before they trust it.

▤ ERC-8183 Jobs

Negotiate, fund, deliver, settle. A signed quote becomes an escrowed job with an on-chain deliverable.

⟀ x402 Self-Funding

Payments settle in $U over x402. The agent pays for what it needs on-chain, no API keys anywhere.

✦ Pieverse LLM

A BNB-native model does the reasoning. Zero-deposit, on-chain metered. The AI itself runs on BNB rails.

◎ MEFAI Signals

The value layer: MEFAI's live signal engine, 40 assets across 6 timeframes, is what the agent actually sells.

100% BNB-native, 100% open source. No closed AI provider, no proprietary API keys. Every layer runs on BNB Chain infrastructure.
06 · Open Source✕
Fork it. Run your own signal agent.

The full agent, its identity, jobs, x402 flow, the MEFAI signal tools and the self-host deploy, is open and MIT-licensed. Clone it, point it at your signal source, and deploy your own seller agent on BNB Chain. The web you are looking at is the repository itself: every node is a real file, indexed by build_codemap.py, and every bright thread is a real import.
